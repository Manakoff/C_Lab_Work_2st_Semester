# 📂 Лабораторные работы по C++ (2 семестр)

Добро пожаловать в репозиторий с лабораторными работами за второй семестр! Этот курс был полностью посвящен углубленному изучению объектно-ориентированного программирования (ООП), разработке графических интерфейсов (GUI) на фреймворке **Qt6** и реализации сетевого взаимодействия с серверами.

---

## 🚀 Ключевые навыки и технологии
* **Парадигмы:** Объектно-ориентированное программирование (ООП) в C++.
* **Фреймворк:** Qt6 (Widgets, Сигналы и слоты, Qt Network).
* **Сетевые технологии:** HTTP/HTTPS запросы к серверам, создание собственного сервера.
* **Инструменты сборки:** CMake / QMake.

---

## 📚 Содержание и изученные темы

В этом семестре упор был сделан на практическое применение ООП и создание полноценных приложений:

### 🔹 Блок 1: Продвинутое ООП и работа со строками
* **Лабораторная «lab string»:** Глубокая работа со строками в C++ (`std::string`, `QString`), кастомные алгоритмы обработки и валидации текстовых данных.
* **Лабораторные 2 и 3:** Разработка собственных классов, реализация инкапсуляции, наследования и полиморфизма. Конструкторы копирования и операторы присваивания.

### 🔹 Блок 2: Разработка графических интерфейсов (GUI) на Qt6
* **Лабораторная 4:** Знакомство с **Qt6 Widgets**. Создание главного окна приложения, работа с кнопками, текстовыми полями и макетами (Layouts).
* **Лабораторная 5 (Вариант 1):** Изучение механизма **сигналов и слотов** (Signals & Slots) для обработки событий пользователя. Проектирование интерактивной логики интерфейса.

### 🔹 Блок 3: Сетевое взаимодействие и работа с серверами
* **Лабораторная 6 (модификации 6_1, 6_2):** Реализация клиент-серверной архитектуры. Отправка сетевых запросов к внешним серверам с использованием модулей Qt. Обработка ответов, работа с API и сериализация данных.

---

## 🛠️ Как запустить проекты локально

1. Убедитесь, что у вас установлен **Qt6** и настроена соответствующая среда (Qt Creator, VS Code с плагинами или Visual Studio с Qt Tools).
2. Склонируйте репозиторий:
   ```bash
   git clone https://github.com
   ```
3. Откройте файл `CMakeLists.txt` или `.pro` файл нужной лабораторной работы в вашей IDE.
4. Соберите и запустите проект.
